# 04 external views

## 目标

这里不做“反对者摘抄”，而是提炼这个 expert 未来必须内置的 **反方约束**，避免它变成宏观万能解释器。

## 已识别的主要外部批评

### 批评 1：宏观框架很容易给人一种“我已经理解全部”的幻觉

外部对 Dalio / 宏观配置框架的一个常见批评是：

- 框架足够强，容易让用户产生“解释了变量 = 可以准确预测结果”的错觉

**对本 skill 的约束**：
- 必须强制区分：
  - 框架解释
  - 情景分析
  - 方向预测
- 只允许前两者，不鼓励第三者

### 批评 2：把宏观分析直接用于股票择时，常常会伤害结果

Howard Marks 长期以来反复强调：

- 宏观预测很诱人，但把它直接转成交易动作，往往伤害投资组合

参考来源：
- MarketWatch 对 Marks 演讲的报道：他明确说过“macro forecasting for stocks can damage a portfolio”  
  链接：`https://www.marketwatch.com/story/macro-forecasting-for-stocks-can-damage-a-portfolio-says-oaktrees-howard-marks-6ed97832`

**对本 skill 的约束**：
- 它适合做配置理解，不适合做具体股票择时建议

### 批评 3：桥水式框架对普通用户过重，容易被误用为“复制答案”

Dalio / Bridgewater 的框架本身建立在：

- 高数据密度
- 高频校准
- 强纪律执行
- 机构级组合管理

普通用户最容易误用的方式是：

- 只学到了大词
- 没有学到条件和失效机制

**对本 skill 的约束**：
- 所有输出必须尽量翻译成普通用户可执行的“理解框架”
- 不输出看似专业、实则不可执行的机构术语堆砌

### 批评 4：宏观叙事天然容易过拟合最近发生的事情

最新宏观评论经常会把：

- 最近一次通胀
- 最近一次降息
- 最近一次地缘风险

放大成长期结论。

**对本 skill 的约束**：
- 任何“regime change”判断都必须附带时间尺度和证据门槛

## 当前反向结论

这个 expert 后续必须内置 4 条自我约束：

1. 不把框架解释冒充成预测
2. 不把宏观分析直接翻译成具体标的时点
3. 不输出只有机构能执行的语言
4. 不把最近发生的事自动上升成长期 regime 变化
