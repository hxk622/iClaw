# 桌面端视觉规范

更新时间：2026-03-18

本规范定义 iClaw 桌面端的品牌视觉基线，覆盖 `Light / Dark` 双主题，并作为 `龙虾商店`、聊天主界面、登录页、设置页和后续金融相关模块的统一视觉来源。

## 目标

- 建立一套有辨识度的金融产品视觉语言，不走常见蓝白灰 SaaS 路线。
- 在 `专业`、`克制`、`现代` 和 `高级感` 之间取得平衡，不做亮金奢华风。
- 让 `Light` 与 `Dark` 共用同一种品牌语气，而不是简单反色。
- 明确区分 `品牌色`、`界面色`、`语义色` 和 `行情色`，避免金融场景颜色冲突。

## 品牌定位

关键词：

- Institutional
- Editorial
- Quiet Luxury
- Precision
- Modern Finance Terminal

中文落点：

- 金融感
- 专业感
- 简洁感
- 不平凡但不过度张扬
- 有机构级秩序感，也有产品辨识度

这套系统不是“奢侈品黑金”，而是“机构级黑白金”。

## 核心原则

### 1. 颜色结构先于单个颜色

视觉系统分为四层：

- `Brand`：黑 / 白 / 灰金，用于品牌识别和高价值强调。
- `Interface`：中性色、背景、边框、分割线，用于搭建界面骨架。
- `Semantic`：成功、警告、错误、信息，用于状态表达。
- `Market`：涨跌、买卖、风险收益，只在金融行情语境里使用。

### 2. 金色只能做强调，不能做底色

金色只用于：

- 当前选中态
- 重点按钮描边或文字
- 品牌徽标、勾边、icon 点缀
- 高价值 badge
- 关键图表主线

金色不要用于：

- 大面积页面背景
- 大段正文
- 高频重复的实底按钮
- 密集数据区底色

建议全局金色占比控制在 `3% - 8%`。

### 3. Light / Dark 共用品牌语气，不共用原始色值

允许共用：

- 品牌灰金方向
- 版式密度
- 字体气质
- 组件层级关系

不建议共用：

- 背景纯度
- 文本亮度
- 描边透明度
- 阴影强度

## 色彩系统

### 品牌主色

品牌主轴不是“纯黑 + 纯白 + 亮金”，而是：

- `石墨黑`
- `象牙白`
- `拉丝灰金`

推荐品牌色：

| Token | 用途 | Light | Dark |
|---|---|---|---|
| `brand.ink` | 品牌主黑、标题、主 CTA 深色版本 | `#171717` | `#F3F0E8` |
| `brand.ivory` | 主背景和高层级留白 | `#F7F5F0` | `#161412` |
| `brand.gold` | 品牌强调 | `#A88C5D` | `#B49A70` |
| `brand.gold-strong` | hover / focus / 高价值高亮 | `#8F7751` | `#C2AA82` |
| `brand.gold-soft` | 金色浅底、选中背景、微光 | `#EAE1D0` | `rgba(180,154,112,0.16)` |

说明：

- `brand.gold` 必须偏灰、偏哑、偏金属，不用亮黄。
- `brand.ink` 在 Dark 下不是纯黑对应纯白，而是暖白，避免冷硬。

### 界面中性色

推荐用中性色搭建骨架，而不是让品牌色承担布局任务。

| Token | 用途 | Light | Dark |
|---|---|---|---|
| `surface.page` | 页面背景 | `#F7F5F0` | `#11100F` |
| `surface.card` | 卡片背景 | `#FCFBF8` | `#191715` |
| `surface.elevated` | 浮层、弹窗、二级面板 | `#FFFFFF` | `#211E1B` |
| `surface.hover` | hover 底色 | `#F1EEE8` | `#26221F` |
| `text.primary` | 主文字 | `#1A1A18` | `#F2EEE6` |
| `text.secondary` | 次级文字 | `#6B655D` | `#B9B0A5` |
| `text.muted` | 辅助信息 | `#9A9288` | `#80786E` |
| `border.subtle` | 弱分割线 | `#ECE7DE` | `#2A2622` |
| `border.default` | 默认边框 | `#DED7CC` | `#3A342E` |
| `border.strong` | 强边框 / 焦点前置边框 | `#C8BEAF` | `#534A42` |

### 语义色

语义色不是品牌色。要能在聊天、支付、登录、设置、安装状态里稳定使用。

| Token | 用途 | Light | Dark |
|---|---|---|---|
| `semantic.info` | 信息、链接、辅助数据提示 | `#315F9E` | `#8EB3E6` |
| `semantic.success` | 成功、通过、已连接 | `#2E6B57` | `#7FC0A9` |
| `semantic.warn` | 警告、进行中、待处理 | `#9A6A22` | `#D1AE74` |
| `semantic.error` | 失败、风险、异常 | `#A14B47` | `#E0A19C` |

规则：

- 语义色全部降饱和，不能抢品牌金的存在感。
- 成功色优先用 `墨绿`，而不是鲜亮荧光绿。
- 信息色优先用 `冷蓝`，避免和金融涨跌色混用。

### 行情色

金融产品必须把 `Semantic` 和 `Market` 分开定义。

推荐规则：

- `market.up`
- `market.down`
- `market.flat`

默认建议：

- A 股场景：
  - `market.up = muted red`
  - `market.down = muted green`
- 美股 / 加密场景：
  - `market.up = muted green`
  - `market.down = muted red`

约束：

- 行情色只用于价格、涨跌幅、K 线、收益对比、买卖方向。
- 不要把 `market.up` 直接当成“成功色”，也不要把 `market.down` 直接当成“错误色”。

## 组件规则

### 侧边栏

- 背景优先使用 `surface.page` 或略深一阶的中性色，不使用金色底。
- 当前选中项建议用 `brand.gold-soft` + `brand.gold` 文字或左侧金色指示条。
- 图标默认使用 `text.secondary`，选中时再切换到 `brand.gold` 或 `text.primary`。

### Tabs

- 未选中：中性色文字 + 轻边框或透明底。
- 选中：使用浅金底或金色描边，不使用高纯度实心金按钮。
- 顶部 tab 是最适合放品牌金的地方之一，适合 `龙虾商店 / 我的龙虾`。

### 卡片

- 卡片是大面积信息容器，优先保持安静。
- 默认卡片：象牙白/暖白底 + 细描边 + 轻阴影。
- hover 只做轻微上浮和边框增强，不做发亮金边。
- 重点卡片可以在角标、状态条、icon 上使用金色，而不是整卡染金。

### 按钮

按钮分三级：

- `Primary`：用深墨色实底，文字用暖白；只在最关键动作使用。
- `Secondary`：浅底 + 细边框，用于补充操作。
- `Brand Accent`：金色描边或浅金底，只用于“精选”“进入商店”“查看详情”等品牌路径。

不要长期大面积使用金色实底主按钮，否则会快速损失克制感。

### 弹窗

- 弹窗背景建议使用 `surface.elevated`。
- 标题和主操作应回到高对比中性色，不要让整窗被金色主导。
- 可在关闭按钮 hover、分节标题 icon、底部安全 badge 上使用少量金色或墨绿。

### Badge / 标签

- `精选`、`官方`、`推荐`：品牌金体系。
- `安全`、`已安装`、`可用`：墨绿体系。
- `风险`、`异常`、`失败`：暗红体系。
- `新功能`、`Beta`、`文档`：冷蓝体系。

## 龙虾商店专项规则

`龙虾商店` 是这套品牌视觉最适合落地的场景之一，但必须避免“电商促销感”。

### 列表页

- 页面整体以留白和中性色为主，不做复杂大背景。
- 顶部 tab 使用浅金选中态，建立品牌记忆点。
- Agent 卡片保持克制：头像、名称、简介、安装按钮四段式即可。
- 安装按钮优先使用中性色主按钮；若需要强调“官方精选”，可在卡片角标或 tab 上用金。

### 详情弹窗

- 头像区域可引入轻微金色 halo 或浅金弧光，但透明度要低。
- `核心功能` 建议使用中性图标 + 少量金色点缀，不做高彩色 icon 墙。
- `适用场景` 区域建议用暖灰浅底，提高阅读完整度。
- 底部 `添加` 按钮可以使用深墨色实底，不建议直接纯金实底。
- `已通过安全合规验证` 类提示建议用墨绿，建立可信感。

### 我的龙虾

- `我的龙虾` 比 `龙虾商店` 更偏工作台语义。
- 已安装状态以中性色和墨绿为主，不需要继续高频使用品牌金。
- 如果有排序、管理、卸载、更新等能力，优先保持专业工具感，而不是“商城感”。

## 字体与排版

当前桌面端以 `PingFang SC` 为主。首版可以继续沿用，但排版语气要往“编辑感 + 终端感”调整。

建议：

- 标题：字重 `600`
- 正文：字重 `400 - 500`
- 小标签：适当增加 tracking，但不要过度全大写
- 正文行高：`1.5 - 1.7`
- 卡片标题避免太花，保持稳重

不建议：

- 过多 oversized 标题
- 过多全大写英文标签
- 大面积细灰字

## 阴影与层次

- 阴影要轻，核心依赖边框和明度分层，而不是厚重投影。
- Light 模式强调柔和边界；Dark 模式强调明暗关系和边框对比。
- 金色的“高级感”主要来自留白、材质感和比例，不来自强发光特效。

## 动效方向

- 动效保持短促、安静、偏产品工具风。
- hover 可轻微上浮和边框增强。
- focus 可使用金色或蓝色的低透明 ring。
- 避免大面积流光、扫光、闪烁金属动画。

## 执行结论

iClaw 的默认视觉方向定为：

- `Light`：象牙白背景 + 石墨黑文本 + 拉丝灰金强调
- `Dark`：炭黑背景 + 暖灰白文本 + 克制暗金强调
- `蓝 / 墨绿 / 暗红` 只作为功能和金融语义辅助

这份文档是后续桌面端页面和组件的视觉基线。新增页面若与本规范冲突，优先回到本规范统一。
