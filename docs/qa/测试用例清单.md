# 垂类金融 AI 桌面端测试用例清单

## 1. 文档说明

本文档用于配套《测试计划2.0》，沉淀垂类金融 AI 桌面端的核心测试用例。测试用例覆盖桌面端基础能力、核心业务流程、金融 AI 专项、RAG 文档解析、异常恢复、安全隔离与合规边界等内容。

建议执行时按以下优先级管理：

1. `P0`：阻塞上线，必须全部通过。
2. `P1`：核心功能高优，原则上需全部通过。
3. `P2`：重要但不阻塞上线。
4. `P3`：体验优化类。

建议每条用例在测试平台或表格中补充以下字段：

1. 用例编号
2. 模块
3. 优先级
4. 前置条件
5. 测试步骤
6. 预期结果
7. 实际结果
8. 是否通过
9. 关联缺陷编号

## 10. 用例维护建议

1. 每个线上缺陷都应尽可能沉淀为固定回归用例。
2. 金融 AI 问答类用例建议同时维护“标准答案”和“关键评分点”两套口径。
3. 对于开放式回答问题，不建议只看文案一致性，应重点看：
   - 是否忠实于输入材料
   - 是否数值正确
   - 是否时间正确
   - 是否越权
   - 是否带来合规风险
4. 建议将测试集拆分为：
   - 基准集
   - 对抗集
   - 合规风险集
   - 缺陷回归集
