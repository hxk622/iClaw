# 02. 平台层与垂类层边界

## 平台层负责什么

平台层负责定义所有垂类都会复用的结构：

- 核心闭环协议：Wiki / Lab / Artifact / Audit
- 本地知识架构
- shell OEM 配置原则
- 垂类扩展阈值
- 可复用对象模型

## 垂类层负责什么

垂类层负责定义某个行业自己的内容和表达：

- 用户画像
- 课程体系
- 场景脚本
- 成果件模板
- 线框 / 交互表达
- 行业成功标准

## 当前 Vertical 01：Financial School

金融垂类当前负责：
- 金融认知课程
- 金融压力实验
- IPS（个人资产投资备忘录）
- 金融专属 wireframe 与验证方案

## 一个简单判断规则

如果某个概念脱离金融仍然成立，它更可能属于平台层。  
如果某个概念只有在金融里才有意义，它更可能属于垂类层。

### 平台层例子
- knowledge object
- scenario
- capstone artifact
- logic audit
- OEM shell entry

### 金融层例子
- CPI / 流动性 / 估值
- FOMO 追高
- 暴跌恐慌
- IPS
- 资产角色定义

## 当前明确不做的平台化动作

- 不做多行业内容包横向扩张
- 不做 Team / Org 学习管理后台
- 不做一次性万能 schema
