# 04. 本地知识架构与 Wiki 模型

## 目标

本地知识层不只是 memory，而是一个可持续编译、更新、审计的知识系统。

## 三层结构

### 1. Raw Sources
原始材料层：
- 课程内容
- 用户上传资料
- 实验记录
- 用户笔记
- 对话摘要
- artifact 历史版本

### 2. Compiled Wiki
结构化知识层：
- 概念页
- 因果关系页
- 用户个人知识页
- 场景页
- 偏差页
- 学习轨迹页

### 3. Schema
约束层：
- 页面类型
- 链接规则
- 冲突标记
- 审计字段
- 版本化方式

## 为什么学习日志必须进入模型

学习日志不是附属信息，而是最关键的“行为证据”。

例如：
- 用户在暴跌模拟中的即时反应
- 用户在热点场景中的犹豫与解释
- artifact 被修改的触发原因

这些日志让系统在未来能做真正的“认知复盘”，而不是只做知识检索。

## 当前建议

v1 默认支持 3 类本地知识对象：
1. 平台知识页
2. 用户个人知识页
3. 学习/实验过程日志

## 当前不做

- 不先做跨组织共享 wiki
- 不先做跨行业统一知识市场
- 不先做所有对象的一次性最终 schema
