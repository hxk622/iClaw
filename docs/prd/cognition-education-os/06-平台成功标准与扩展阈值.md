# 06. 平台成功标准与扩展阈值

## 平台成功标准

平台是否成立，不看“支持多少行业”，而看：

> 同一套 Wiki / Lab / Artifact / Audit 底层结构，能否完整支撑 Financial School 的真实闭环。

## 最小通过条件

1. 金融学校能在同一套底层上完成：
   - 知识映射
   - 压力实验
   - 成果件更新
   - 逻辑审计
   - 规则修正
2. 这些动作不是分散模块，而是共用同一协议/模型
3. admin-web 能配置 shell 入口，而不需要重写客户端壳
4. 系统能保留“修正前 / 修正后”的可追踪差异

## 为什么不是先看第二垂类

因为第二垂类是否能做出来，不足以证明平台成立。  
真正的平台成立，应该先在最高压力场（金融）跑通最完整闭环。

## 允许启动第二垂类的阈值

只有满足以下条件之一，才允许正式讨论 Vertical 02：

1. Financial School 闭环已稳定跑通
2. 平台层与金融层边界在文档和实现上都清晰
3. artifact、scenario、knowledge object 的抽象已被真实使用验证
4. 第二垂类的需求不是“想象出来的”，而是来自已知用户/业务压力

## 当前不允许的扩展理由

- 只是因为“平台听起来更大”
- 只是因为“另一个行业也能讲认知”
- 只是因为 shell 已经可以换皮

这些都不是启动第二垂类的充分理由。
