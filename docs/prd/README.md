## 文档组织原则

- `README.md`：该 PRD 的总览、导航、关键结论
- `01-*.md`：定位、目标、用户、边界
- `02-*.md`：模块、流程、用户旅程
- `03-*.md`：验收、验证、风险、里程碑

新增 PRD 时，建议按同样结构建立独立目录，避免把多个产品方向混在一份文档里。
