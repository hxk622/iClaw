# 18. 核心页面低保真 Wireframe

## 目标

本文件在上一份 shell 级 wireframe 的基础上，继续定义学习系统内部关键页面的低保真结构。

这些 wireframe 只定义：
- 信息区块
- 布局顺序
- 主 CTA
- 页面间跳转关系

不定义最终视觉风格。

---

## 1. 学习首页 / Dashboard

### 页面目标
让用户一进入学习系统就知道：
- 现在在哪个阶段
- 下一步该做什么
- 最近哪里出了问题
- IPS 还差什么

### 低保真结构

```text
┌──────────────────────────────────────────────────────────────┐
│ Header: 学习 / 当前阶段 / 进度 / 账户                         │
├──────────────────────────────────────────────────────────────┤
│ 阶段状态卡                                                    │
│ [阶段：建立模型] [课程进度 3/8] [IPS 62%] [待复盘 1]          │
├──────────────────────────────────────────────────────────────┤
│ 主任务区                                                      │
│ 你下一步最该做的事：                                          │
│ [继续课程：价格、价值与估值]                                  │
├──────────────────────────────────────────────────────────────┤
│ 当前认知盲点                                                  │
│ - 容易把热点上涨误当确定性                                    │
│ - 风险边界描述仍然模糊                                        │
├──────────────────────────────────────────────────────────────┤
│ 学习路径预览                                                  │
│ 诊断 -> 课程 -> 实验 -> IPS -> 复盘                          │
│          ↑ 当前在这里                                          │
├──────────────────────────────────────────────────────────────┤
│ 最近一次实验反馈                                              │
│ FOMO 追高：你在“参与条件未明确”时选择了追入                   │
│ [去复盘] [修改 IPS]                                           │
└──────────────────────────────────────────────────────────────┘
```

### 页面 CTA
- 继续课程
- 去实验室
- 修改 IPS
- 查看复盘

## 4. 实验室页 / Simulation Lab

### 页面目标
让用户在模拟中做选择、承担后果、识别偏差、触发复盘。

### 低保真结构

```text
┌──────────────────────────────────────────────────────────────┐
│ 顶部：实验室 / 场景 1：FOMO 追高                              │
├──────────────────────────────────────────────────────────────┤
│ 场景背景                                                      │
│ 某热门主题 10 天上涨 25%，全网都在讨论                         │
├──────────────────────────────────────────────────────────────┤
│ 当前冲突点                                                    │
│ “你会因为怕错过而上车，还是先判断参与条件？”                   │
├──────────────────────────────────────────────────────────────┤
│ 决策面板                                                      │
│ ( ) 立即追入                                                  │
│ ( ) 小仓位试错                                                │
│ ( ) 暂不参与，先写参与条件                                    │
│ ( ) 明确放弃                                                  │
│ [提交决策]                                                    │
├──────────────────────────────────────────────────────────────┤
│ 结果反馈（提交后展开）                                        │
│ - 你选择了：立即追入                                          │
│ - 暴露偏差：把上涨误当确定性                                  │
│ - 长期代价：买入价格过高，风险收益比恶化                      │
├──────────────────────────────────────────────────────────────┤
│ 复盘问题                                                      │
│ - 你为什么要参与？                                            │
│ - 如果明天跌 20%，你的预案是什么？                            │
│ [开始复盘] [去改 IPS]                                         │
└──────────────────────────────────────────────────────────────┘
```

### 页面 CTA
- 提交决策
- 开始复盘
- 去改 IPS
- 回补课程

## 页面流转关系

```text
学习首页
  -> 学习地图
  -> 当前课程
  -> 最近实验
  -> IPS 工作台

学习地图
  -> 课程详情

课程详情
  -> 小测
  -> 实验室
  -> IPS 工作台

实验室
  -> 复盘页
  -> IPS 工作台
  -> 补课

复盘页
  -> IPS 工作台
  -> 返回学习首页
```

## 页面级判断标准

如果一个页面：
- 不能明确告诉用户下一步做什么
- 不能把课程、实验、IPS 串起来
- 不能让错误导向系统升级

那它就还不是这套产品需要的 wireframe。
