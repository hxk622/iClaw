# 06. 认知实验室

## 核心定位

认知实验室是 v1 的核心，不是装饰模块。

它的作用不是“让学习更有趣”，而是：

> 让用户在低成本环境中犯错、承受压力、复盘并修正底层假设。

## 为什么必须做成核心机制

- 真正的认知改变来自“知行冲突”被暴露
- 现实市场中的痛感成本太高，必须在模拟环境中先经历
- 只有经历情境压力，用户才会认真写预案，而不是停留在口头理解

## v1 至少要有的 3 个场景

### 场景 1：FOMO 追高
- 市场主题快速上涨
- 周围都在晒收益
- 用户需要判断：这是自己理解范围内的机会，还是情绪驱动的追逐？

### 场景 2：深度回撤 / 恐慌
- 资产出现 20%–30% 回撤
- 用户需要调用既定风险边界和应对预案
- 系统观察其是否能坚持概率逻辑

### 场景 3：横盘 / 策略漂移
- 市场迟迟不给反馈
- 用户容易怀疑系统、频繁切换策略
- 系统观察其是否能维持长期框架

## 机制建议

- 场景触发后，允许用户做判断
- 系统展示后果与偏差来源
- 若缺乏预案，强制进入反思/静默/重写步骤
- 每次实验后都要把反馈写回 IPS

## 游戏化原则

游戏化不是积分糖衣，而是“认知疫苗”。

可接受的游戏化：
- 场景闯关
- 决策后果反馈
- 认知信用分
- 复盘完成度

不应成为核心的游戏化：
- 单纯签到
- 无意义金币
- 与认知无关的勋章收集
