# 05. 认知课程系统

## 目标

课程系统不是“知识点仓库”，而是帮助用户搭起金融因果骨架。

## v1 建议模块

建议先做 6–8 个核心模块：

1. 金融的本质：风险定价与跨时空资源配置
2. 收益、风险与概率：为什么高收益不能脱离高不确定性
3. 估值与价格：价格为何围绕价值波动
4. 周期与流动性：宏观变量如何传导到资产
5. 资产配置与相关性：为什么分散不是平均分配
6. 行为偏差：FOMO、损失厌恶、锚定、过度自信
7. 预案与纪律：为什么系统先于判断
8. 复盘与升级：如何从一次错误中更新系统

## 每个课程模块的标准结构

每个模块至少包含：

- 一段第一性原理讲解
- 一张模型图解
- 一个小型思维测验
- 一个真实/模拟应用场景

## 课程设计原则

- 用因果链条组织内容，而不是按术语堆砌
- 用“为什么”驱动，而不是只给“是什么”
- 每个模块都必须能落到一个判断动作或风险边界
