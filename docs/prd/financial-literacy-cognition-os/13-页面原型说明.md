# 13. 页面原型说明

## 目标

本页给出 v1 四个关键页面的原型说明：

1. 首页
2. 课程页
3. 实验室页
4. IPS 页

这些说明不是视觉稿，而是页面级产品原型定义，供后续 IA、交互和 UI 设计使用。

---

## 一、首页原型说明

### 页面目标
首页不是资讯页，而是“当前操作面板”。

用户打开首页时，应该立刻知道：
- 我现在在哪个认知阶段
- 我下一步该做什么
- 我最近一次错在哪里
- 我的 IPS 还缺什么

### 页面区块

#### 1. 顶部状态卡
- 当前认知阶段
- 当前连续学习进度
- IPS 完成度
- 最近一次实验结果摘要

#### 2. 主任务区
动态显示一个最重要 CTA：
- 继续当前课程
- 进入待完成实验
- 修改 IPS
- 查看复盘建议

#### 3. 当前认知盲点
- 系统识别出的 1–2 个高优先级问题
- 如：FOMO、风险边界模糊、资产角色理解错误

#### 4. 学习路径缩略图
- 当前所在模块
- 下一个模块
- 已完成节点

#### 5. 最近复盘记录
- 最近一次实验错误
- 对应 IPS 修改建议

### 原型原则
- 首页必须聚焦行动，不做资讯流
- 首页不展示“推荐股票/热点追踪”
- 首页的每个内容块都要指向一个下一步动作

---

## 二、课程页原型说明

### 页面目标
帮助用户理解一个具体金融原理，并把它放入更大的认知图谱中。

### 页面区块

#### 1. 模块头部
- 模块标题
- 该模块解决什么问题
- 所属学习阶段
- 预计耗时

#### 2. 第一性原理讲解区
- 核心概念解释
- 反常识点
- 该原理在现实中的意义

#### 3. 模型图解区
- 用结构图/因果图展示概念如何互相作用
- 可标出变量、传导路径、结果

#### 4. 思维检查区
- 不是背诵题
- 更偏判断题、推理题、解释题

#### 5. 场景应用区
- 把本模块知识放进一个小场景
- 例如：流动性变化如何影响估值与风险偏好

#### 6. 关联到 IPS 的提示
- 提醒用户：这节内容会影响 IPS 中哪一部分

### 原型原则
- 每一课都必须从“概念”落到“判断”
- 每一课都要能连接到实验室或 IPS
- 不做纯文章阅读体验

---

## 三、实验室页原型说明

### 页面目标
让用户在模拟环境中做决策、承受后果、识别偏差并回写系统。

### 页面结构

#### 1. 场景背景区
- 当前年份/周期/市场气氛
- 用户持仓设定（可虚拟）
- 当前冲突点是什么

#### 2. 决策面板
- 用户从几个方案中做出选择
- 或填写自己会怎么处理

#### 3. 情绪提示层
- 提醒用户：现在最容易出现什么认知偏差
- 也可以隐藏，作为复盘后揭示项

#### 4. 结果反馈区
- 展示短期与长期后果
- 强调不是“答错了”，而是“你的假设会导向什么代价”

#### 5. 偏差解释区
- 指出用户用了什么错误模型
- 对应课程中哪部分知识没真正掌握

#### 6. 下一步动作区
- 去补某个课程模块
- 去修改 IPS 某条规则
- 进入复盘页

### 原型原则
- 实验页必须有压力感，但不能做成投机游戏
- 实验结果必须导向认知修正，而不是积分奖励
- 每个实验都必须能追溯到具体认知错误

---

## 四、IPS 页原型说明

### 页面目标
帮助用户把认知系统写成可审计、可迭代、可复盘的个人规则文档。

### 页面区块

#### 1. IPS 概览区
- 当前版本号
- 完成度
- 最近更新时间
- 一句话策略摘要

#### 2. 资产角色区
- 我为什么配置这些资产
- 每类资产的作用是什么

#### 3. 风险边界区
- 最大可接受回撤
- 时间周期
- 不允许突破的原则

#### 4. If-Then 规则区
- 如果暴跌 x%，我怎么做
- 如果热点暴涨，我怎么判断是否参与
- 如果长期没反馈，我怎么避免策略漂移

#### 5. 证据与假设区
- 当前规则建立在什么假设上
- 哪些证据会推翻这些假设

#### 6. 历史版本与修改说明
- 最近改了什么
- 为什么改
- 是哪次实验或复盘触发了修改

### 原型原则
- IPS 页必须像“工作台”，不是静态作文页
- 用户应能看到规则、边界、假设和修改历史
- 页面要支持持续迭代，而不是一次填写结束

---

## 页面之间的关系

```text
首页
  -> 课程页
  -> 实验室页
  -> IPS 页

课程页
  -> 小型场景应用
  -> 引导进入实验室
  -> 引导修改 IPS

实验室页
  -> 结果反馈
  -> 复盘
  -> IPS 修改

IPS 页
  -> 回看历史
  -> 响应实验反馈
  -> 形成下一轮行为规则
```

## 页面级设计原则总结

- 首页负责“调度”
- 课程页负责“建模”
- 实验室页负责“打脸与暴露偏差”
- IPS 页负责“沉淀与升级系统”

四个页面必须共同构成一个闭环，而不是四个孤立页面。
