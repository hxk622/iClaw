# 11. 信息架构 IA

## 目标

信息架构要服务于一个核心原则：

> 用户不是来“刷内容”，而是来逐步安装一套金融认知操作系统。

因此 IA 不能按传统内容站的“文章分类”来组织，而要按**认知闭环**来组织。

## 一级结构

建议 v1 采用 6 个一级信息区：

1. **首页 / Home**
2. **学习地图 / Curriculum**
3. **认知实验室 / Simulation Lab**
4. **IPS 工作台 / IPS Workspace**
5. **复盘记录 / Review History**
6. **个人中心 / Profile & Progress**

---

## 一级结构说明

### 1. 首页 / Home
**作用：** 承接用户当前状态，告诉他“下一步该做什么”。

**核心内容：**
- 今日/当前学习任务
- 当前所处认知阶段
- 最近一次实验反馈
- IPS 进度提示
- 核心提醒：当前最值得修正的认知盲点

**不应承载：**
- 大量资讯流
- 市场热点流
- 文章堆砌式内容广场

---

### 2. 学习地图 / Curriculum
**作用：** 呈现完整认知课程图谱。

**核心内容：**
- 课程模块列表
- 每个模块的目标、难度、状态
- 图解、测验、场景任务入口
- 推荐学习顺序

**设计原则：**
- 按认知能力和因果逻辑组织，不按资产品类随意切分
- 模块之间要有先修关系与阶段感

---

### 3. 认知实验室 / Simulation Lab
**作用：** 在模拟环境中让用户暴露错误假设，并触发修正。

**核心内容：**
- 场景列表
- 场景背景设定
- 用户决策步骤
- 后果反馈
- 偏差解释
- 复盘入口

**设计原则：**
- 每个实验都要服务于一个明确认知错误
- 实验结果必须能回写到 IPS 或复盘记录

---

### 4. IPS 工作台 / IPS Workspace
**作用：** 帮用户沉淀自己的投资认知系统。

**核心内容：**
- IPS 当前版本
- 资产角色说明
- 风险边界
- If-Then 规则
- 修改建议与版本历史

**设计原则：**
- IPS 是工作台，不是一次性问卷
- 内容应可持续编辑、比较、审计

---

### 5. 复盘记录 / Review History
**作用：** 记录用户的错误、变化与认知升级轨迹。

**核心内容：**
- 实验失败记录
- IPS 改写历史
- 常见偏差标签
- 模块学习前后变化

**设计原则：**
- 帮助用户看到“自己怎么错、怎么改、怎么变强”
- 强化系统升级感，而不是制造羞耻感

---

### 6. 个人中心 / Profile & Progress
**作用：** 展示用户当前画像、学习阶段和进度。

**核心内容：**
- 认知阶段
- 已完成模块
- 实验通过情况
- IPS 完成度
- 关键能力画像

**设计原则：**
- 用于组织学习，不用于泛社交或排行榜炫耀

---

## 推荐导航方式

### 顶层导航建议
- 首页
- 学习地图
- 实验室
- IPS 工作台
- 复盘
- 我的

### 首页上的主 CTA
根据用户状态动态切换：
- 继续当前课程
- 进入待完成实验
- 补写/修改 IPS
- 查看复盘建议

---

## 对象关系图

```text
用户
  -> 入门诊断结果
  -> 学习路径
  -> 课程模块
       -> 图解
       -> 小测
       -> 场景任务
  -> 实验室场景
       -> 决策记录
       -> 偏差反馈
  -> IPS
       -> 版本历史
       -> 风险边界
       -> If-Then 规则
  -> 复盘记录
       -> 修改建议
       -> 成长轨迹
```

## IA 核心判断标准

如果一个页面或模块：
- 不能帮助用户理解金融因果
- 不能帮助用户暴露错误认知
- 不能帮助用户形成或修正 IPS

那它在 v1 里就不该是核心入口。
