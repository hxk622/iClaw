# 08. 学习路径与进度机制

## 目标

进度机制的作用不是制造上瘾，而是帮助用户在 90 天内走完认知安装闭环。

## v1 路径框架

### 阶段 1：诊断与校准
- 识别用户当前认知误区
- 生成初始学习路径

### 阶段 2：认知输入
- 课程 + 图解 + 小测
- 建立基础模型

### 阶段 3：情境受压
- 进入认知实验室
- 在模拟中暴露错误和情绪反应

### 阶段 4：IPS 外化
- 把判断系统写成文档
- 建立书面化规则

### 阶段 5：复盘升级
- 在失败和修改中形成稳定结构

## 进度展示建议

可以有：
- 学习地图
- 模块完成状态
- 当前正在修炼的认知能力
- 已完成的 IPS 关键节点

不应把进度机制做成：
- 纯连续打卡工具
- 与认知质量脱钩的排行榜
- 鼓励刷时长而非刷理解的数值系统
