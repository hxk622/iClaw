# 02. 信息架构与三栏模型

## 一级入口

左侧主菜单新增：

- 知识库

进入知识库后，主界面切换为固定三栏工作区。

## 三栏结构

### 第一栏：对象导航

职责：

- 展示当前知识对象列表
- 按知识层级切换
- 提供搜索、过滤、导入、新建入口

固定三个 tab：

#### Tab 1：Raw / 素材

展示原始输入对象，来源包括：

- 本地上传
- Web clip
- 页面保存
- 划词摘录
- 视频时间戳摘录
- 对话沉淀
- 转写结果

#### Tab 2：Ontology / 本体图谱

展示经过编译后的结构化知识对象：

- 图谱文档
- 子图
- 节点集
- 关系集

#### Tab 3：Output / 成果

展示所有可复用 artifact：

- Memo
- 专家资产
- 小红书内容
- 公众号内容
- PPT
- Doc
- 规则卡
- 其他二创输出

---

### 第二栏：对象视图

职责：

- 根据第一栏选中的对象，展示该对象的结构化视图
- 不同对象类型使用不同 viewer

#### Raw 视图

至少包含：

- 标题
- 来源
- 标签
- 摘要
- 正文 / 摘录 / 片段
- 来源元信息
- 推荐动作（编入图谱 / 继续提炼 / 加入对话）

#### Graph 视图

至少包含两种模式：

- 页面视图
- 图谱视图

图谱视图目标是 Obsidian 风格的节点-连线关系图；Graphify 作为候选渲染内核进行集成验证。

#### Output 视图

至少包含：

- 成果标题
- 成果正文 / 预览
- 引用素材
- 引用图谱
- 推荐动作（继续二创 / 发布 / 反哺图谱）

---

### 第三栏：对话

职责：

- 始终复用 iClaw 现有 Chat Surface
- 选中对象只作为上下文源，不产生第二套聊天系统

行为要求：

- 进入知识库时，输入框默认是空的
- 选中对象后，以 context 形式注入第三栏，而不是把说明文字直接塞进输入框
- 用户可以选择“在主对话中打开”，把当前对象切换到全宽主会话里继续处理

## 典型用户流

### 1. 浏览器摘录流

1. 用户在网页中滑选内容
2. 插件执行 pick / save
3. 内容进入 Raw / 素材
4. 客户端第一栏可见新素材
5. 第二栏展示素材详情
6. 第三栏围绕该素材继续对话

### 2. 图谱提炼流

1. 用户选中一条或多条 Raw
2. 系统执行编译
3. 生成 Graph 对象
4. 第一栏 Graph tab 出现图谱对象
5. 第二栏展示页面视图或关系图
6. 第三栏用图谱上下文做问答/总结/生成

### 3. 成果生产流

1. 用户基于 Raw / Graph 发起生成
2. 系统生成 artifact
3. 第一栏 Output tab 出现成果对象
4. 第二栏预览成果
5. 用户继续修改、发布或反哺图谱

## 设计约束

- 第一栏负责导航，不承载重内容编辑
- 第二栏负责主要视图，不承载主聊天
- 第三栏是协作入口，不承载复杂对象编辑器
- 三栏必须始终同时存在，避免在知识工作流中频繁切页
