# 01. 产品定位与范围

## 产品定义

iClaw 知识库是一个 local-first 的知识工作台，用来承接原始输入、编译关系图谱、沉淀成果件，并把这些对象持续作为 AI 协作上下文。

它不是传统资料库，也不是单纯 memory 容器，而是一条知识生产流水线：

- Raw 负责进入
- Graph 负责编译
- Output 负责产出
- Chat 负责协作与反馈

## 用户目标

知识库需要服务的核心任务：

1. 快速捕获有价值的原始内容
2. 在客户端中持续整理、提炼、连接关系
3. 生成可复用的成果件与专家资产
4. 在后续对话中节省 token，提高理解准确率

## V1 范围

### 包含

- 固定三栏知识库 Shell
- 第一栏三 tab：Raw / Graph / Output
- Raw 的真实对象化与真实列表
- 第二栏对象视图
- 第三栏上下文驱动 Chat
- 浏览器插件作为 Raw ingest 入口之一
- 本地存储 + 本地查询 + 本地去重

### 不包含

- 多组织 / Team / Org 管理后台
- 通用跨行业 Schema 的一次性完美抽象
- 复杂协同编辑
- 成熟版图谱画布编辑器
- 企业知识权限系统

## 关键边界

### 1. 先深做知识工作流，不做泛内容广场

知识库优先解决的是「个人知识采集 - 编译 - 产出 - 对话」闭环，不先扩展成内容分发平台。

### 2. 插件属于入口，不属于中心

浏览器插件不作为知识库主界面。主界面始终在 iClaw 客户端。

### 3. Graph 是编译层，不是装饰层

图谱不是一个好看的视图，而是 Raw 经过结构化和关系抽取后的中间知识层。

### 4. Output 是 artifact，不是临时聊天记录

Output 只收纳有复用价值的对象，例如：

- 研究 memo
- 专家人格资产
- 二创内容
- 文档/PPT
- 规则卡/备忘录

## V1 实施顺序

### Phase 1：Raw 真实化

目标：

- 第一栏 Raw 列表读真实数据
- 第二栏展示真实素材详情
- 第三栏基于真实 Raw 做上下文对话

### Phase 2：Graph 数据化

目标：

- 引入图谱对象结构
- 第二栏展示图谱页面视图与关系视图
- 让 Raw -> Graph 成为实际流水线，而不是静态示意图

### Phase 3：Output 对象化

目标：

- 把 memo、专家、二创件、文档件都纳入统一 artifact 结构
- 支持 Output 反向引用 Raw 与 Graph

## V1 非目标提醒

当前阶段不要过早做：

- 跨行业模板平台化
- B 端组织学习后台
- 复杂图谱协同编辑
- 插件与桌面端自动登录桥的重型方案
