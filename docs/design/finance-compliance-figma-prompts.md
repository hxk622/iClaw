# 理财客金融合规 Figma Prompts

更新时间：2026-04-19

本文件汇总理财客金融合规相关界面的 Figma / AI 设计 prompt，便于直接复制给设计师或图形生成工具使用。

## 1. 聊天页弱降级提示条

```text
为「理财客」桌面端聊天页设计一个金融合规弱降级提示条。

目标：
- 当回答被降级为“研究参考口径”时，在聊天区顶部或当前回答上方显示一条轻量提示。
- 不打断对话，不喧宾夺主，但必须足够清晰，让用户知道这不是直接投资建议。

产品背景：
- 品牌：理财客 / 财富管理导向 / 中文界面
- 风格：克制、可信、偏金融终端，不要像营销弹窗
- 语气：审慎、专业、低情绪

文案参考：
- 当前回答已按研究参考口径展示。请结合前提、风险与失效条件自行判断，不应直接视为投资建议。

设计要求：
- 适配桌面端聊天主界面
- 需要给出 normal / hover / compact 三种状态
- 颜色使用金融场景可信配色，避免高饱和警告红
- 视觉优先级低于报错卡，高于普通说明文案
- 给出与消息气泡、输入框、任务结果卡的相对位置

输出：
- Figma 设计稿说明
- 组件结构
- 间距、字号、色板、圆角、边框建议
- 如何在多轮对话中避免重复打扰
```

## 2. 聊天页高风险说明卡

```text
为「理财客」桌面端聊天页设计一个“高风险金融回答说明卡”。

目标：
- 当模型输出接近 actionable advice，但产品策略要求只按 research-only 展示时，
  用一张说明卡包住这段回答或放在回答上方，提示用户这是研究参考，不是执行建议。

需要表达的信息：
- 这是 AI 生成内容
- 当前内容已按研究参考口径展示
- 请关注前提、风险、失效条件
- 可提供“展开原文”或“继续查看分析细节”的入口，但不要默认鼓励操作

风格：
- 金融终端风格
- 专业、克制、可信
- 不要像风控拦截报错页，也不要像营销横幅

请输出：
- 桌面端设计稿
- 展开前 / 展开后 两种状态
- 卡片标题、说明、次级操作入口、免责声明区位置
- 与普通 assistant 消息气泡的组合关系
```

## 3. 定时任务结果页免责声明区块

```text
为「理财客」桌面端“定时任务结果页”设计一个金融免责声明与风险提示区块。

背景：
- 场景是晨报、周报、市场日报、自动研究摘要
- 任务结果可能包含市场观点，但通知摘要已经被降噪
- 结果页里需要有统一的风险提示，不影响阅读主内容

目标：
- 将“本回答由AI生成，仅供参考，请仔细甄别，谨慎投资。”以稳定、低打扰但清晰可见的方式呈现
- 同时容纳“生成时间 / 数据时间 / 来源”这类辅助信息

设计要求：
- 适配结果页卡片式布局
- 与“任务名称 / 执行状态 / 执行结果 / 失败原因”兼容
- 分为 success / failure / no-disclaimer 三种布局状态

请输出：
- 设计稿
- 组件层级
- 视觉规范
- 内容密度高时的折叠策略
```

## 4. 通知中心详情抽屉

```text
为「理财客」桌面端通知中心详情抽屉设计“金融风险提示”区域。

目标：
- 当一条通知来自金融任务结果时，在详情抽屉中展示一块风险提示区域
- 与现有“任务名称 / 执行状态 / 执行时间 / 执行结果 / 失败原因”区块协调一致

设计要求：
- 右侧抽屉布局
- 一眼可见，但不抢主标题
- 能承载短免责声明与较长解释文案
- success 和 error 场景都兼容

请输出：
- 设计稿
- 信息层级
- 与现有抽屉内容的编排关系
- 风险提示颜色和样式建议
```

## 5. admin-web 审计页

```text
为「理财客」后台 admin-web 设计“金融合规审计页”。

页面目标：
- 给运营、法务、产品看“为什么这次显示/没显示免责声明”
- 提供概览统计、列表筛选、单条事件详情

页面模块：
1. 概览统计卡片
   - 总事件数
   - 免责声明命中率
   - degraded 数量
   - blocked 数量
2. 分布模块
   - 按渠道
   - 按输出分类
   - Top 命中规则
3. 事件列表
4. 详情面板
   - app / session / channel
   - input classification
   - output classification
   - risk level
   - reasons
   - disclaimer text
   - metadata JSON

风格：
- 管理后台风格
- 信息密度高
- 清晰、冷静、可审计
- 不做花哨图表，优先可读性

请输出：
- 页面级设计稿
- 列表 + 详情双栏布局
- 筛选区组件
- 统计卡片风格
- 风险等级与状态标签系统
```

## 6. 高风险阻断卡

```text
为「理财客」桌面端聊天页设计一个“高风险金融执行场景阻断卡”。

触发场景：
- 用户要求直接执行买卖、仓位调整、下单、申赎等动作
- 系统策略需要阻断，而不是仅做弱提示

目标：
- 明确告诉用户当前请求属于高风险金融执行场景
- 说明系统当前只支持研究参考，不直接支持执行性结论
- 如有需要，可提供“继续查看研究分析”或“返回修改问题”的引导

设计要求：
- 比弱降级提示更强，但仍保持专业克制
- 不做恐吓式红色警告页
- 与聊天布局兼容

请输出：
- 阻断卡设计稿
- 主文案 / 次文案 / 操作入口结构
- 与普通错误卡、余额不足卡的差异化建议
```

## 7. 使用建议

- 优先把 prompt 直接交给设计师或 Figma AI，先出低保真结构稿，再迭代视觉细节。
- 聊天页相关 prompt 建议同时附上当前聊天页面截图和品牌色板。
- admin-web prompt 建议附上现有后台页面截图，保持列表 / 详情 / 筛选的风格一致。
