# Knowledge Graph View V1

## 目标

定义 iClaw 知识库中 Ontology / 本体图谱 的 V1 展示方式。

结论：

- 图谱对象属于第一栏的 Ontology / 本体图谱 tab
- 图谱的主要展示位置在第二栏
- 第二栏至少支持“页面视图”和“图谱视图”两种模式
- `graphify` 作为候选渲染项目进行集成验证

参考候选项目：
- `https://github.com/safishamsi/graphify/tree/v3`

## 1. 设计原则

### 1. 图谱首先是知识关系，不是视觉装饰

图谱视图必须可追溯到：

- 哪些 Raw 导致了这个图谱
- 哪些节点来自哪些证据
- 哪些边代表什么关系

### 2. 页面视图优先于复杂画布编辑

V1 不追求重型图谱编辑器，而优先保证：

- 有真实节点
- 有真实边
- 有真实证据来源
- 有可阅读的图谱页面视图

### 3. Obsidian 风格是视觉目标，不是功能边界

目标效果：

- 深色背景
- 节点云
- 节点连线
- 局部聚类
- 可聚焦当前子图

但 V1 不要求实现完整 Obsidian 式交互编辑能力。

## 2. 第二栏 Graph 两种模式

### A. 页面视图

用于结构化阅读，内容包括：

- 图谱标题
- 图谱摘要
- 核心节点列表
- 核心关系列表
- 证据来源列表
- 最近更新时间

### B. 图谱视图

用于关系浏览，内容包括：

- 节点
- 边
- 当前焦点节点
- 证据高亮

## 3. Graphify 的角色

Graphify 在 V1 中定义为：

- 图谱渲染候选内核
- 用于把 `GraphNode + GraphEdge` 映射成可视节点关系图

但 Graphify 不替代：

- 图谱 schema
- 图谱编译器
- 证据链模型
- iClaw 的对象协议

也就是说：

> Graphify 负责“画”，iClaw 负责“图谱是什么”。

## 4. V1 必做项

- 第二栏 Graph 页面视图
- 第二栏 Graph 图谱视图
- 节点-边真实数据绑定
- 选中图谱对象后第三栏可拿到图谱上下文

## 5. V1 非必做项

- 拖拽编辑节点
- 自定义布局编辑
- 多图谱协同操作
- 大规模图谱性能优化
- 完整 graph workspace

## 6. 推荐实现顺序

### Step 1

先定义 GraphDocument / GraphNode / GraphEdge 的数据协议

### Step 2

先在第二栏把页面视图做成真数据 viewer

### Step 3

引入 Graphify 做关系图渲染验证

### Step 4

把节点点击 -> 第二栏详情 / 第三栏对话上下文打通

## 7. 验收标准

如果一个图谱对象满足以下条件，则 V1 成立：

- 第一栏 Ontology / 本体图谱 tab 可以看到真实对象
- 第二栏页面视图能看到真实摘要与关系说明
- 第二栏图谱视图能看到真实节点和连线
- 节点和边可以追溯到 Raw 证据
- 第三栏对话能基于该图谱继续协作
