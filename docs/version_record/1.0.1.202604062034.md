## 13. 待发热修：支付网关显式“继承平台 / OEM 独立”模式

- 状态：`code-ready / not deployed`
- 时间：`2026-04-07 00:08:30 CST`
- 范围：
  - `control-plane`
  - `admin-web`
- root cause：
  - 当 OEM scope 未真正落成独立配置时，页面只显示“当前来源=继承平台”，但缺少显式模式切换，容易误判为“已独立配置”
  - 在这种状态下修改平台网关会同步反映到 OEM，行为本身正确，但 UI 不够清晰，且缺少“恢复继承平台”显式操作
- 修复点：
  - `admin-web` OEM 支付网关新增显式 `Gateway Mode`：
    - `继承平台`
    - `使用 OEM 独立网关`
  - 继承平台模式下表单禁用，保存时删除 OEM app-scope 网关配置并恢复继承
  - 独立模式下保存仅更新当前 OEM scope，不影响平台和其它 OEM
  - `control-plane` 新增 app-scope 网关配置删除能力，并覆盖对应服务测试
- 本地验证：
  - `pnpm check:control-plane`
  - `pnpm --filter @iclaw/control-plane test`
  - `pnpm build:admin`
