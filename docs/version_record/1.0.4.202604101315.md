## 2. 变更摘要

- 本次发布目标：推进 `1.0.4.202604101315`，将 Windows 升级主链路切到 `installer-driven force upgrade`，并完成厚包安装性能优化。
- 用户可见变化：
  - Windows 命中强更后改为自动下载完整安装包、自动拉起安装器、升级后恢复现场
  - Windows 厚包继续保留，但 bundled runtime 已做瘦身优化
  - 安装包与下载页继续按 `iclaw` / `licaiclaw` 两个品牌分别发布
- 风险最高的点：
  - `1.0.4` 期间存在同一公开版本号下多次补改，流程追溯不干净
  - QA 反馈安装器在 `96%` 长时间停留，根因一度缺乏机器侧日志
  - 未签名 Windows 包仍可能被 `SmartScreen` 阻断

## 7. 风险点

- 同一公开版本号下存在补丁流，违反“一次发布只认一个不可变基线”的原则
- QA 机器缺少安装阶段结构化日志上传能力，导致 `96%` 问题根因证据不足
- 未签名 Windows 包仍会触发 `SmartScreen` / 信誉风险
- 默认 offline WebView2 会让安装包接近三倍膨胀，且没有证明它是主因，不能作为默认策略
