## 7. 风险点

- 全量覆盖 `app` schema 后，旧 `access_tokens / refresh_tokens / device_sessions` 失效属于预期，QA 需重新登录
- 如果源环境与 prod 在账号核心表结构上不一致，回灌会失败；因此必须先以相同 schema 完成恢复，再回灌账号数据
- 对象存储全量 mirror 会覆盖 prod 现有对象，必须保证 prod 侧保留回滚备份
- 若重发桌面安装包，用户重装后本地历史可能丢失；本次默认不把“保本地历史”作为硬约束
