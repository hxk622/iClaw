# Docs Maintenance Checklist

更新时间：2026-04-12

## 提交前检查

每次新增或移动文档前，至少检查以下事项：

- 运行 `pnpm docs:check`

### 1. 归类正确

- 是否放入正确主题目录
- 是否误放在 `docs/` 根目录
- 是否被错误放入 `archive/`

### 2. 命名正确

- 是否为小写英文
- 是否使用 `-` 分词
- 是否带上合适后缀：
  - `-architecture.md`
  - `-design.md`
  - `-standard.md`
  - `-implementation-plan.md`

### 3. 索引已更新

- 是否加入对应子目录 `README.md`
- 若属于高频入口，是否加入 `docs/README.md`

### 4. 链接已更新

- 是否还残留旧路径
- 是否还残留本机绝对路径
- 是否存在失效的相对路径

### 5. 内容完整

- 是否包含更新时间
- 是否包含目标 / 范围 / 结论
- 是否明确说明该文档适用于谁

## 移动文档后检查

- 搜索旧路径引用并修正
- 确认 `docs/` 根目录没有重新堆散文件
- 检查相邻文档的交叉引用是否仍然成立

## 长期维护要求

- 新文档先分类，再落盘
- 目录索引比单个文件更重要
- 不允许把临时文档长期留在 `archive/`
