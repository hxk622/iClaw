# P0 Payment Cases

## Case: 充值中心套餐来自后端配置

ID:
- `PAY-E2E-003`

脚本:
- `tests/payment/recharge-package-flow.test.mjs`

目标:
- 验证充值中心先展示套餐页，且套餐卡片真实可用

前置:
- `desktop web shell` 运行在 `http://127.0.0.1:1520`
- `control-plane` 运行在 `http://127.0.0.1:2130`
- Chrome CDP 已开启
- 存在可登录测试账号

步骤:
1. 登录 desktop
2. 从头像/额度入口打开 `充值中心`
3. 观察套餐选择页
4. 点击 `立即充值`

断言:
- 首屏是套餐页，不是二维码页
- 套餐卡片数量大于 0
- 点击继续后进入支付页

截图点:
- 套餐页
- 支付页

## Case: 支付二维码布局首屏完整可见

## Case: 过期二维码可刷新重建

ID:
- `PAY-E2E-006`

脚本:
- `tests/payment/expired-refresh.test.mjs`

目标:
- 验证二维码过期后出现遮罩，并可刷新生成新二维码

前置:
- 同上

步骤:
1. 登录并进入支付页
2. 让当前订单进入过期态
3. 点击刷新二维码

断言:
- 出现过期遮罩
- 出现刷新按钮
- 刷新后二维码链接变化

截图点:
- 过期遮罩
- 刷新后的二维码
