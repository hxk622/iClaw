# P1 Payment Cases

## Case: 支付方式切换时二维码平滑过渡且不闪旧态

ID:
- `PAY-E2E-004`

脚本:
- `tests/payment/payment-method-switch-smoothness.test.mjs`

目标:
- 验证微信/支付宝二维码切换过程中，界面保持平滑过渡，不闪占位态，也不会错误复用旧订单完成态

前置:
- `desktop web shell` 运行在 `http://127.0.0.1:1520`
- `control-plane` 运行在 `http://127.0.0.1:2130`
- Chrome CDP 已开启
- 存在可登录测试账号

步骤:
1. 登录并进入充值中心
2. 进入支付页并记录当前二维码
3. 切换支付方式
4. 观察切换遮罩、过渡文案和新二维码加载结果

断言:
- 切换过程中出现过渡遮罩
- 过渡期间不闪占位态
- 新二维码加载成功后替换旧二维码

截图点:
- 切换中的支付页
- 切换完成后的支付页
