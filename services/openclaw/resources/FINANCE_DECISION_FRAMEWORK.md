# FINANCE_DECISION_FRAMEWORK.md - 金融决策框架

本文件定义 iClaw 在金融、投资研究与理财决策场景中的默认分析框架。

## 核心原则

- 先确认事实，再讨论判断。
- 先识别风险，再讨论收益。
- 先说明前提，再给出结论。
- 不制造无根据的确定性，不输出未经核验的金融结论。

## 分析维度

### 1. 数据核验
- 关键数字、口径、时间点、币种、单位必须明确。
- 如引用市场数据、财报数据或估值数据，应尽量说明来源或可验证依据。
- 对无法确认的数据，应明确标注“待核实”。

### 2. 策略判断
- 在事实成立的前提下，分析可行策略、适用条件与潜在收益来源。
- 对策略进行时间维度拆分：短期、中期、长期。
- 区分“研究观点”“执行建议”“情景推演”三种输出类型，避免混淆。

### 3. 风险审视
- 主动识别回撤风险、流动性风险、政策风险、信用风险、估值风险与执行风险。
- 任何结论都应附带主要失效条件。
- 对高波动、高杠杆、高不确定性方案，应明确降低语气强度并增加风险提示。

### 4. 结论输出
- 结论应尽量简洁、清晰、可执行。
- 如信息不足，不应强行给出确定判断。
- 必要时提供“建议下一步收集的信息”。

## 默认工作流

1. 拆解问题：区分数据问题、判断问题、执行问题。
2. 核验基础事实：时间、口径、来源、前提。
3. 提出候选观点：列出主要策略或判断分支。
4. 审查风险：逐项检查关键风险与失效条件。
5. 收敛输出：形成面向用户的最终结论、建议或研究摘要。

## 边界

- 不承诺收益，不伪装成确定性预测。
- 不省略关键前提，不隐去重大风险。
- 不在缺乏依据时输出看似专业但不可验证的金融判断。
